# V1.5 F9 — One-Click Demo Mode · Audit

`src/lib/demo/orchestrator.ts` seeds a synthetic `aguila-demo` company (idempotent via `ensureDemoCompany()`) and a fresh `DEMO-XXXXXX` tráfico, then walks a 12-step lifecycle with ~4s spacing between steps (≈48s runtime; ~90s including DB latency). Steps fire `workflow_events` and, where the schema allows without foreign-key gymnastics, write real rows to `classification_sheets`, `invoices`, and `quickbooks_export_jobs`. `pece_payments`, `anexo_24_export_jobs`, and `mve_alerts` are fired as workflow events only because they require pedimento FKs we do not synthesise. `/admin/demo` is admin/broker-gated (silver glass single-button UI) with a side panel that polls `/api/demo/status/[runId]` every 3s; the in-memory `demoRuns` Map tracks progress and is the intentional limitation — on a serverless cold start to a different lambda the poll returns `unknown` and the UI falls back to the last known client-side state. `/api/demo/reset` purges synthetic tráficos + child rows + QB + invoice + mve rows scoped to `company_id = aguila-demo`, and is idempotent. Telemetry rides on `interaction_events` as `demo_run_started`, `demo_run_completed`, and `demo_reset` with event payload, keeping the locked TelemetryEvent union unchanged. CLAUDE.md cross-domain list and `INTERNAL_GROUPS` nav gain `/admin/demo` under a new "Interno" group. **Deferred:** SSE streaming (polling is sufficient for demo cadence), real PDF generation for DODA/Carta Porte inside the demo chain (out of scope for F9), cross-lambda run persistence (would require a `demo_runs` table), and a Telegram cancellation window on the first step (all effects are synthetic so irreversibility is a non-issue). **Test delta:** +5 tests (287 → 292) covering `buildInitialRun` shape, `makeTraficoRef` format, full orchestrator walk with all 12 events + telemetry, domain-row insertion (classification + invoice + QB), and in-memory run lookup.
