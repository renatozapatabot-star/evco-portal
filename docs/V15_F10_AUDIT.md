# V1.5 F10 — Operator Performance Dashboard · Audit

`src/lib/operators/metrics.ts` computes per-operator metrics for a date range from five null-safe sources: `operators` (canonical roster, active+company-scoped), `operator_actions` (drives `traficosHandled` as distinct `target_id` where `target_table='traficos'`, `errorRate` as fraction of actions matching `/error|reject|failed/`, `lastActiveAt` as max `created_at`), `workflow_events` (drives `avgCycleHours` as the average hours between the earliest `supplier_confirmed`/`entrada_received` and the latest `semaforo_verde` per assigned tráfico), `traficos.assigned_to_operator_id` (the grouping key for cycle + MVE), `mve_alerts` (tráficos without an alert in range count as clean for `mveComplianceRate`), and the optional `document_classifications` table for `classificationAccuracy` (null when the table is absent — F14 will populate it). `GET /api/admin/operadores/metrics?from&to` is admin/broker-only, defaults to the current UTC month, scopes every query to `session.companyId`, and fires a `metadata.event = 'operator_metrics_viewed'` row into `interaction_events` without widening the TelemetryEvent union. `/admin/operadores` ships a silver-glass header with native date-range inputs, a sortable seven-column table (mono on every numeric column + percentages + hours + timestamps) with 60px row hit targets, an empty state, and a 375px card-per-operator fallback driven by a scoped `@media` block. Row click routes to `/admin/operadores/[id]`, a server component that renders an operator header plus a 2-column layout of per-action-type counts and the last 50 authored `operator_actions`. Nav gets a new "Equipo" group under `INTERNAL_GROUPS`; CLAUDE.md Cross-domain list gains both routes. **Deferred:** per-operator goals/targets, time-of-day heatmap, classification accuracy depends on F14's `document_classifications` table being provisioned, peer-comparison charts, per-operator streak + duelo integration, drill-through from the detail page into the underlying tráfico timelines, and real-time refresh via Supabase Realtime. **Test delta:** +5 tests (292 → 297) covering the empty-operator path, action-driven metrics (traficos, errors, last active), the workflow-event cycle-hours average, MVE compliance + classification accuracy, and the `formatPct`/`formatHours` helpers.
