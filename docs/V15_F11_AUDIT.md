# V1.5 F11 — Cliente Self-Service Portal Upgrade

Cliente inicio (`/inicio`) now renders a 3-tab cockpit for the client role: Mis tráficos activos, Documentos, Notificaciones. Each active tráfico is a silver-glass card with ref, status pulse, ETA and last-event summary that links straight through to `/traficos/[id]/trace`. Documentos lists `expediente_documentos` grouped per tráfico with signed-download links, and Notificaciones renders the last 30 `workflow_events` with Spanish cliente-friendly labels for the top 15 event kinds (semáforo, warehouse, pedimento, documents, lifecycle) and a Title-Cased fallback for the rest. Non-client roles transparently redirect to `/`. RLS handles the hard company isolation; `src/lib/cliente/dashboard.ts` layers an explicit `company_id` filter on top and batches the tráfico→events/documents joins with `.in()` to avoid N+1s. Telemetry fires `metadata.event = 'cliente_inicio_viewed'` via `logOperatorAction` on mount. Deferred: real-time subscription, per-tráfico chat thread, supplier reply surface — all queued for F12+. Test delta: +1 file, +6 assertions (cliente-event-labels), 297 → 303.
